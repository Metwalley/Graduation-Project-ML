# Meeting Script: Technical Decisions & Handoff

## 1. The Problem Space & Solutions Considered

"يا جماعة، لما جيت أبدأ شغل على الـ Chatbot كان قدامي 3 طرق، وكل طريق كان فيه مشكلة كبيرة:"

### ❌ Option 1: Custom Model from Scratch (التدريب من الصفر)
*   **الوصف:** أجيب موديل زي BERT أو GPT-2 وأدربه على داتا طبية من الصفر (Fine-tuning).
*   **العيوب القاتلة:** 
    1.  **يحتاج داتا ضخمة:** إحنا معندناش آلاف المحادثات الطبية الموثقة بالعربي عشان الموديل يتعلم منها.
    2.  **يحتاج GPU قوي:** تدريب موديل محتاج سيرفرات بآلاف الدولارات، وده مش متاح لينا.
    3.  **النتيجة مش مضمونة:** ممكن بعد كل التعب ده الموديل يهلفط (Hallucinate) ويدي معلومات غلط، وده خطر في مجال طبي/تعليمي.

### ❌ Option 2: Cloud APIs (زي OpenAI GPT-4 أو Gemini)
*   **الوصف:** نستخدم API جاهز وندفع اشتراك.
*   **العيوب القاتلة:**
    1.  **التكلفة:** كل سؤال بفلوس. لو فتحنا التطبيق للناس، الفاتورة ممكن تكسر الميزانية في يومين.
    2.  **الخصوصية:** بيانات المستخدمين بتتبعت لسيرفرات بره، وده ضد مبادئ التطبيقات الطبية/النفسية.
    3.  **التحكم (Latency & Reliability):** معتمدين على سرعة النت عندهم، ولو السيرفر وقع، التطبيق كله بيقع. (وده حصل معانا فعلاً مع Gemini لما كان بيقطع الإجابات).

### ✅ The Smart Solution: Local LLM + RAG (Ollama)
*   **الوصف:** نستخدم موديل قوي ومفتوح المصدر (Llama 3.2) ونشغله عندنا على السيرفر (Local) باستخدام أداة اسمها Ollama، وبندعمه بمعلوماتنا الخاصة (RAG).
*   **ليه هو أنسب حل؟**
    1.  ** ببلاش (No Cost):** مفيش فواتير API، الموديل شغال على جهازنا.
    2.  **Privacy 100%:** الداتا مابتطلعش بره السيرفر بتاعنا.
    3.  **Offline Capable:** شغال حتى لو النت قطع (مهم جداً في المناقشة).
    4.  **Control:** إحنا اللي بنتحكم في الـ Safety والدقة (زي ما عملنا مع الفلتر الطبي).
*   **العيوب البسيطة:**
    1.  **Hardware:** محتاجين جهاز فيه RAM معقولة (8GB+) عشان يشغل الموديل. (بس ده محلول، الموديل خفيف).
    2.  **Initial Setup:** التسطيب محتاج خطوات زيادة عن مجرد API key، بس إحنا عملناها Dockerized عشان نسهلها.

## 2. Technical Handoff (إزاي هيشغلوه؟)

"عشان نربط البوت بالـ Backend بتاعكم، الموضوع بسيط جداً لإننا عملناه Microservice معزولة:"

### الطريقة: Docker Container 🐳
"أنا هديكم **Docker Image** (أو Dockerfile)، وده عبارة عن صندوق مقفول فيه كل حاجة (الـ Code، المكاتب، وموديل Ollama نفسه)."

### الخطوات للـ Backend Team:
1.  **Run Container:** هتاخدوا الأمر ده وتشغلوه عندكم على السيرفر:
    ```bash
    docker-compose up -d
    ```
2.  **API Integration:** البوت هيشتغل كـ **REST API** على بورت `8001`.
3.  **Endpoint:** انتوا هتبعتوا الـ Request كده:
    *   **URL:** `http://localhost:8001/chat`
    *   **Method:** `POST`
    *   **Body:**
        ```json
        {
            "text": "ابني مش بيركز في الواجب، أعمل إيه؟"
        }
        ```
    *   **Response:** هيرد عليكم بـ JSON فيه الإجابة:
        ```json
        {
            "response": "يمكنك اتباع الخطوات التالية...",
            "sources": [...]
        }
        ```

"يعني حرفياً، انتوا بتكلموا API عادي جداً، بس الـ API ده شغال عندنا Local مش على Cloud."
